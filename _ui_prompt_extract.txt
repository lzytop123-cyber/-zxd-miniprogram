
知行岛自习室 · 小程序 UI 优化
Cursor UI 优化 Prompt 文档 · 面向大学生用户
清新薄荷绿风格 · 微信原生小程序



一、项目背景与品牌定位
产品名称：知行岛自习室
品牌理念：取自&quot;知行合一&quot;，倡导专注、踏实的学习态度。小程序是大学生预约自习座位、记录学习时长的主要工具。

目标用户：大学生为主，追求安静高效的学习环境
核心场景：预约自习座位 → 蓝牙开门入座 → 学习计时 → 查看学习报告

设计气质：
清新自然，像绿植环绕的安静自习室
简洁克制，不花哨，帮助用户保持专注
有温度感，不冰冷，适合陪伴学习的产品调性

配插画图标（使用 SVG 图标库，如 IconFont 绿色系图标）

骨架屏 Skeleton
加载时使用骨架屏替代菊花 loading
骨架色：#E8F5EC（浅绿灰），动画：shimmer 从左到右
Toast 提示
胶囊样式，圆角 20px，#1C2B20 背景，白色文字
不使用微信默认 showToast 样式

三、各页面 UI 细化要求

📍 页面 1：首页 pages/home/index — 门店列表

页面结构
顶部 Header：品牌 Logo + 「知行岛自习室」文字 + 右侧消息图标
搜索栏：圆角搜索框，placeholder「搜索门店名称或地址」，带定位图标
Banner 轮播：门店宣传图，圆角 12px，指示点用绿色小点
「附近门店」区块标题 + 门店卡片列表

门店卡片设计
封面图：宽度 100%，高度 140px，圆角 16px 16px 0 0
信息区（padding 12px）：
  第一行：门店名称（15px 粗体）+ 营业状态 Tag（「营业中」绿色 / 「已打烊」灰色）
  第二行：地址（12px 灰色）+ 距离（12px 绿色）
  第三行：剩余座位数（用绿色徽标突出「剩 12 座」）+ 最低价格（「¥3/小时起」）
卡片间距：margin-bottom: 12px

Cursor 指令
请重写 pages/home/index.wxml 和 .wxss，实现以上结构。门店卡片封装为独立组件 components/store-card/。

📍 页面 2：门店详情 pages/store/detail

页面结构
顶部：门店大图（高度 220px）+ 返回按钮（左上角白色圆形背景图标）
基础信息卡片：门店名、地址、营业时间、简介
价格区块：各卡种价格列表（小时卡/次卡/月卡），横向滚动卡片
设施标签：Wi-Fi、打印、饮水机等，用浅绿色 Tag
底部固定按钮：「立即预约」主绿色大按钮

Cursor 指令
保持页面原有数据绑定，只重写样式。封面图使用 image mode=&quot;aspectFill&quot;。

📍 页面 3：选座 pages/store/seats

座位状态色
可选座位：背景 #D8F3DC，边框 #52B788，文字 #2D6A4F
已占用：背景 #F5F5F5，边框 #CFD8DC，文字 #8C9BA5，cursor 禁用
我选中：背景 #52B788，边框 #2D6A4F，文字 #FFFFFF，右上角绿色勾

布局细节
座位用网格布局（grid），每排根据实际座位数自适应
每个座位格子：宽高 44px，圆角 8px，字号 11px 显示座位编号
顶部加图例说明：三个小色块 + 文字（可选 / 已占 / 已选）
底部显示「已选：A-03」+ 「下一步」按钮

Cursor 指令
座位数据来自 seatList 数组，字段包含 id、status（available/occupied/selected）、label。wx:for 循环渲染，bindtap 切换选中态。

📍 页面 4：预约 pages/booking/index

时间段选择
横向 scroll-view，不换行
Chip 样式：高度 36px，圆角 18px，padding 0 16px
未选中：白色背景 + #52B788 描边；选中：#52B788 背景 + 白字
已过时段：灰色背景 + 删除线文字

卡种选择
卡片式单选列表，每项：卡种名（加粗）+ 有效期说明 + 价格（右对齐，绿色大字）
选中态：绿色 2px 边框 + 右上角绿色勾形角标
推荐卡种加「推荐」橙色小 Tag

Cursor 指令
时段数据 timeSlots[]，卡种数据 cardTypes[]。选中状态用 selectedTime 和 selectedCard 变量控制，setData 更新。

📍 页面 5：确认订单 pages/booking/order

页面结构
订单信息卡片：门店名、座位号、日期、时段、时长
优惠券选择行：「使用优惠券」→ 展示可用张数 / 去选择 &gt;
期限卡抵扣行：若有期限卡，显示可抵扣金额
费用明细：原价 / 优惠 / 实付，用分隔线区分
底部：实付金额（大字绿色）+ 「确认支付」主按钮

Cursor 指令
金额计算在 JS 中完成，页面只做展示。支付按钮调用 wx.requestPayment，加 loading 态防重复点击。

📍 页面 6：入座 pages/checkin/index

核心布局
中央大圆形区域：显示门店名 + 座位号，底部「蓝牙开门」主按钮
「蓝牙开门」按钮：直径 120px 圆形，#52B788 背景，白色门图标，阴影突出
剩余时长：进度环（圆形进度条）+ 中间显示「剩余 2h 30min」
底部操作：「提前离座」次级按钮（描边样式）

Cursor 指令
蓝牙功能已有逻辑，只重写 UI 部分。进度环用 canvas 或 CSS conic-gradient 实现。

📍 页面 7：学习报告 pages/report/index

数据展示
今日学习时长：大数字（如「4h 23min」）+ 进度条对比目标
本周日历热力图：7 格，绿色深浅表示当天学习时长
本月柱状图：轻量 bar chart，用 echarts-for-weixin 或手写 canvas
排行榜列表：头像（圆形）+ 昵称 + 本周时长；第1名显示皇冠图标（🏆）

Cursor 指令
图表数据来自接口，先用 mock 数据开发。排行榜自己排第几用绿色底色高亮。

📍 页面 8：个人中心 pages/profile/index

顶部用户信息区
绿色渐变背景（#2D6A4F → #52B788），圆角仅底部 24px
用户头像：圆形 64px，白色 2px 描边
昵称：白色 18px 加粗
积分 + 余额横向排列，白色半透明小卡片

功能入口宫格
2列宫格或列表，图标统一用线性风格（推荐 Iconify phosphor 系列）
图标底色：浅绿圆形背景 #D8F3DC，图标色 #52B788
入口项：钱包、我的订单、期限卡、优惠券、积分明细、邀请有礼

Cursor 指令
头部区域用 view 模拟，不用 cover-view。宫格用 grid 布局，2列等宽。

3. 不引入第三方 UI 库，只用微信原生组件

本页面具体要求：[粘贴本文档对应页面的「页面结构」和「细节要求」内容]

当前代码如下：
[粘贴 .wxml 代码]
[粘贴 .wxss 代码]

请：
① 重写 .wxss 样式，如需要也调整 .wxml 结构

③ 确保兼容 iOS 底部安全区

技术约束（每次都需告诉 Cursor）：
微信原生小程序，.wxml + .wxss，不使用 npm 包
基准屏幕宽度 375px，单位使用 rpx（1px = 2rpx）
图片使用 &lt;image&gt; 标签，mode=&quot;aspectFill&quot;
滚动使用 &lt;scroll-view&gt;，横向加 scroll-x=&quot;true&quot;
颜色统一在 app.wxss 定义 CSS 变量，各页面引用变量
