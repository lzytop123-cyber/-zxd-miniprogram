# 知行岛自习室 DESIGN.md

## 1. 设计方向

知行岛是面向线下自习室用户的预约与学习服务。界面应传达 **安静、温暖、可靠、专注**，让用户能快速完成找门店、选座、预约、入座和管理会员权益等高频动作。

以暖白和自然绿为主：绿色用于确认、可用状态和主行动；留白、浅色分区与克制阴影建立舒适感。参考实体门店会员体验的亲和与品质感，但不复刻任何现有品牌的标识、排版或图形资产。

优先级：

1. 预约和入座状态必须一眼可辨。
2. 座位、时间、价格与权益必须准确、易扫读。
3. 装饰只服务于安静、温暖的氛围，不能干扰操作。

## 2. 视觉令牌

| 语义 | 值 | 用途 |
| --- | --- | --- |
| 主绿 | `#52B788` | 主按钮、选中、可用座位、正向状态 |
| 深绿 | `#2D6A4F` | 标题、深色横幅、强调文字 |
| 浅绿 | `#D8F3DC` | 选中背景、成功提示、弱强调标签 |
| 页面暖白 | `#F7F8F4` | 页面底色；避免冰冷纯灰 |
| 卡片白 | `#FFFFFF` | 卡片、底部操作栏、弹层 |
| 主文字 | `#1C2B20` | 标题、正文、关键数据 |
| 次要文字 | `#6F7D73` | 辅助说明、次要元信息 |
| 分割线 | `#E8ECEA` | 分组、弱边界 |
| 警示金 | `#D58B17` | 待支付、临近到期、活动信息 |
| 错误红 | `#C95C5C` | 取消、异常、不可用状态 |

使用色彩表达状态，而不是装饰。一个页面内最多一个高饱和主操作色；警示金和错误红只用于对应状态。

## 3. 排版与间距

- 小程序正文优先系统字体：`-apple-system, BlinkMacSystemFont, "PingFang SC", "Microsoft YaHei", sans-serif`。
- 标题不要使用衬线、花体或全大写英文；中文以清晰自然为先。
- 页面标题：`40rpx / 700`；区块标题：`30–32rpx / 600–700`；正文：`26–28rpx / 400`；辅助文字：`22–24rpx`。
- 使用 8 的倍数间距：`16 / 24 / 32 / 40 / 48rpx`。页面横向边距默认 `24rpx`，卡片内边距默认 `28–32rpx`。
- 数字、价格、剩余座位和时长使用更强字重；单位和说明降低层级。

## 4. 布局原则

- 用户端采用单列、清晰分区的移动优先布局；先展示当前预约或最短路径，再展示活动与推荐内容。
- 首页顺序建议为：当前预约/入座状态 → 核心入口（订座、入座）→ 会员权益 → 附近门店 → 活动与公告。
- 门店、套餐、订单均采用白色圆角卡片；卡片之间保留 `24rpx` 间距，避免连续边框堆叠。
- 预约页始终让「日期、时段、座位、价格、提交」处于清晰的线性流程中；提交操作固定在底部安全区上方。
- 后台以信息效率为先：保留 Element Plus 的表格、筛选与表单模式，只统一品牌色、统计卡片和状态标签，不把移动端的大圆角、渐变横幅照搬到后台。

## 5. 组件规范

### 卡片

- 圆角 `28–32rpx`，白底，阴影使用 `0 4rpx 24rpx rgba(28,43,32,.06)`。
- 优先用留白、弱阴影和浅分割线建立层级；不要用厚描边、黑色阴影或玻璃拟态。
- 重要卡片可在顶部使用 4–6rpx 深绿色细色条，不能大面积渐变铺满所有卡片。

### 按钮

- 主按钮：主绿底白字，最小高度 `88–96rpx`，圆角 `24rpx`，字重 `600–700`。
- 次按钮：白底绿边或浅绿底深绿字；不得与主按钮争夺视觉权重。
- 危险操作使用红色文字或二次确认，不使用醒目的红色大面积主按钮。
- 点击反馈仅用轻微透明度或 `scale(.98)`，时长 120–180ms。

### 标签与状态

- 标签为圆角胶囊，字号 `22rpx`，使用浅色背景 + 深色文字。
- 座位状态必须同时具备文字/图例和颜色：可选（绿）、已选（深绿或绿边高亮）、已预约（警示金）、使用中（低饱和深绿）、不可用（灰）。
- 不要只靠颜色传达座位状态；选中状态必须有边框、图标或文字变化。

### 表单与选择器

- 输入框和选择器高 `88rpx` 左右，圆角 `16rpx`，背景使用卡片白或很浅的暖白。
- 选日期、时段、座位时，选择结果要在当前页面持续可见。
- 支付前必须汇总展示门店、座位、时段、时长、价格、卡券抵扣和实付金额。

### 会员与权益

- 会员卡保持温暖、克制的绿色层次，可使用深绿至主绿的单向渐变。
- 数字权益（余额、积分、剩余次数）优先大数字 + 小标签；不要使用金融产品式的复杂图表。
- 优惠活动使用警示金作小面积点缀，避免与主绿竞争。

## 6. 图片与图标

- 门店图片优先使用真实、明亮、安静、有桌椅与自然光的空间照片；避免过度滤镜、空旷样板间或人物特写抢占注意力。
- Banner 首选活动信息与门店氛围的结合，文字覆盖区域要有足够对比度。
- 图标使用同一套线性或圆角图标风格，线宽一致；不要混用彩色 3D 图标、Emoji 和不同来源的拟物图标。

## 7. 禁止项

- 不使用大面积纯黑、霓虹渐变、赛博风、强玻璃拟态或高频动效。
- 不使用超过两种品牌强调色；不为每个卡片分配独立渐变。
- 不让活动 Banner 压过「订座」与「入座」两个核心动作。
- 不在用户端堆叠密集表格、复杂筛选或后台式仪表盘。
- 不复制 Starbucks 或任何参考品牌的 logo、专属字体、文案、图形资产或识别性布局。

## 8. 实施提示

现有小程序的 `#52B788`、`#2D6A4F`、`#D8F3DC` 和圆角卡片已与本规范一致。后续改版应优先复用 `miniprogram/app.wxss` 中的语义变量；若新增颜色或尺寸，先补充为变量，再在页面中使用，避免硬编码和风格漂移。
